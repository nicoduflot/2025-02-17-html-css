Problèmes CSS Courants

Le CSS par défaut des navigateur date des premiers navigateur, et est essentiellement utile pour de l'impression directe.

Ce CSS est inadapté sur beaucoup de niveaux

  • Le calcul des largeurs des éléments est basé sur le contenu des éléments et non pas l'extérieur des bordures des éléments
  • L'accessibilité, la police de caractère utilisée, avec empatement n'est pas accessible
  • La taille par défaut de la police n'est pas adaptée à la diffusion sur les appareils modernes (smartphone, tablettes, crans modernes)
  • L'interlignage n'est pas adapté (pour les supports cités précédemment)

Le calcul des largeur d'éléments

On change le comportement par défaut du navigateur box-sizing: content-box en intervenant sur tous les éléments de la page grâce au sélecteur *, indiquant la propriété suivante box-sizing: border-box

Les largeur des éléments se calculeront désormais depuis l'extérieur des bordures et non depuis la largeur du contenu de l'élément.

La police de caractère

Pour rendre une site accessible, il est préconisé d'utiliser une police de caractère sans empatement : sans-serif

La taille des polices

De base, les navigateur utilisent le cadratin em comme mesure pour les polices (et aussi pour les marges).

Le problème réside dans le fait que le em se calcul par rapport à la taille déclaré dans l'élément parent.

style="font-size: 2em;

style="font-size: 2em;

Pour remédier à cela, on va préferer utiliser le cadratin racine, son aventage : il se calcul par rapport à la taille de la police racine (:root)

style="font-size: 2rem;

style="font-size: 2rem;

L'interligne

De base l'interligne est trop petit sur les navigateur, il faudra l'augmenter pour rendre le contenu lisible, principalement pour les appareils avec un petit écran.

Type inline

Il s'agit principalement des balises permettant de mettre en forme des parties d'un texte dans un élément de type paragraphe.

Le gras et l'italique peuvent chacune être représentés par deux balise, qui visuellement rendent le même résultat, mais dont l'impact en référencement sera différent.

Pour ne pas impacter le référencement, on utilisera <b> (bold) pour le gras et <i> (italic) pour l'italique.

Si les éléments mis en valeurs doivent jouer sur le référencement, on utilisera <strong> (strong) pour le gras et <em> (emphasis) pour l'italique.

Pour changer l'apparence du texte, on peut biffer, souligner (ATTENTION : par défaut, ON NE SOULIGNE PAS LE TEXTE EN WEB : pour ne pas le confondre avec les liens), passer un marqueur fluo, écrire des formules chimiques : H2SO42-

Des balises déprécié (a ne plus utiliser)

La balise <font> servait a changer la police, la taille et la couleur d'un morceau de texte mais la balise, même si elle est encore interprétée par les navigateurs n'est plus a utiliser. On utilisera à la place une balise <span> à une classe CSS pour faire ces changements.

Les images

La balise de base pour appeler une image <img />. Il faut au minimum l'attribut src qui indique la source de l'image chargée.

En accessibilité, il faudra ajouter l'attribut alt, c'est un texte alternatif qui doit décrire l'image. Si il s'agit d'une image de décoration, on laisse l'attribut vide.

En référencement, alt est important, ainsi que le nom du fichier image. Si on veut capitaliser sur une image et l'associer à une légende qui améliorera sont référencement, il est important de l'encapsuler dans une balise <figure>

Figure

Figure est une balise qui permet d'associer une légende à une image

Le fait d'indiquer que par défaut toutes les images s'affiche jusqu'au maximum de leur largeur permet d'éviter les débordements lorsqu'on les affiche sur un écran plus petit que leur largeur.

Picture

En adaptatif, on peut ajouter à la largeur maximale la performance d'affiche ( = rapidité) le choix de la meilleure image adaptée à la largeur de l'écran, avec <picture>
